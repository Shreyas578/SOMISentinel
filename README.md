## 🎯 What is SOMI Sentinel?

SOMI Sentinel is a cutting-edge **reactive DeFi guardian** engineered specifically for the Somnia L1 ecosystem. In the volatile world of decentralized finance, milliseconds matter. Traditional monitoring tools rely on RPC polling—periodically checking the blockchain state—which introduces inherent delays of 12 seconds or more. This lag can be the difference between a successful rescue and a catastrophic liquidation.

**SOMI Sentinel solves this by leveraging Somnia Native Reactivity's push-based architecture.** Instead of "asking" the blockchain for updates, SOMI Sentinel maintains a persistent WebSocket connection to a Somnia node. Using the native `somnia_subscribe` method, it receives instant, same-block notifications for critical events via Reactive Data Streams (RDS).

**Key Features:**
- ⚡ **Sub-second latency** - Events appear in UI < 1 second after on-chain execution.
- 🔴 **Zero polling** - Uses native `somnia_subscribe` for autonomous RDS push notifications.
- 📊 **Real-time charts** - Live candlestick visualization of position health.
- 🐋 **Whale detection** - On-chain intelligence for large-scale movements.
- 🛡️ **Autonomous protection** - Instantly reacts to volatility in the data stream.
- 📡 **Live Data Stream** - Monitors an automated simulation of market conditions.

## ⚡ How Somnia Reactivity Is Used

SOMI Sentinel is built entirely around the **Somnia Reactivity** paradigm, moving away from legacy HTTP/RPC polling.

### 1. Unified Event Stream (`somnia_subscribe`)
The backend service establishes a persistent WebSocket connection to the Somnia node. By calling `somnia_subscribe` with the `RDS` parameter, we receive a real-time stream of all relevant smart contract events (deposits, price changes, whale transfers) directly from the Somnia Native Data Stream.

// wss://dream-rpc.somnia.network/ws
ws.send(JSON.stringify({
  method: "somnia_subscribe",
  params: ["RDS", { eventFilter: { address: [MOCK_TOKEN, LENDING_POOL, GUARDIAN] } }]
}));
```

### 2. Same-Block Delivery
Somnia Reactivity ensures that when a transaction is mined, the associated events are pushed to our service *within the same block cycle*. This guarantees that the UI state and event log are always synchronized perfectly, reflecting the actual state of the ledger without lag.

### 3. Cross-Environment Pipeline
1.  **On-Chain**: Solidity contracts emit events during state transitions.
2.  **Infrastructure**: Somnia nodes detect these events and push them via WebSocket.
3.  **Middleware**: Our TypeScript Reactivity Service decodes the binary logs using ABIs.
4.  **Frontend**: The decoded data is broadcast to the React dashboard, triggering instant UI updates and toast notifications.

### 4. Zero RPC Overhead
Because we rely on a single push-connection, we avoid the hundreds of redundant HTTP requests generated by traditional polling architectures, making the application significantly more efficient and responsive.

## 🚀 Running the Project

### Prerequisites
- Node.js 18+
- MetaMask browser extension
- STT (Somnia Testnet tokens) — [Get from faucet](https://testnet.somnia.network/)

### 1. Clone & Install

```bash
git clone https://github.com/YOUR_USERNAME/SOMISentinel
cd SOMISentinel

# Install all dependencies
cd contracts        && npm install && cd ..
cd reactivity-service && npm install && cd ..
cd frontend         && npm install && cd ..
```

### 2. Deploy Smart Contracts

```bash
cd contracts
cp .env.example .env
# ✏️ Edit .env — add your PRIVATE_KEY (MetaMask wallet funded with STT)
npm run deploy
# Copy the printed addresses!
```

### 3. Configure Contract Addresses

After deployment, update two files with the printed addresses:

**`reactivity-service/.env`**:
```env
MOCK_TOKEN_ADDRESS=0x...
LENDING_POOL_ADDRESS=0x...
GUARDIAN_ADDRESS=0x...
```

**`frontend/.env.local`**:
```env
NEXT_PUBLIC_MOCK_TOKEN=0x...
NEXT_PUBLIC_LENDING_POOL=0x...
NEXT_PUBLIC_GUARDIAN=0x...
NEXT_PUBLIC_REACTIVITY_WS=ws://localhost:3001
```

### 4. Start the Reactivity Service

```bash
cd reactivity-service
cp .env.example .env   # fill in addresses
npm run dev

cd contracts
npx hardhat run scripts/simulate-fluctuations.ts --network somnia
#to start the data stream
```

This service connects to `wss://dream-rpc.somnia.network/ws` and subscribes to contract events via Somnia Native Reactivity (`somnia_subscribe`).

### 5. Start the Frontend

```bash
cd frontend
npm run dev
# Open http://localhost:3000
```
